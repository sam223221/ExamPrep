# DOCUMENT.md — `data/`

## What lives here

The generated content the app serves: study guides (`guides/`) and the curated
MCQ bank (`mcqs/`), the archived original MCQ bank (`mcqs-archive/`), plus the
runtime-generated ChromaDB store and slide cache (both gitignored).

## Subdirectories

| Path            | Owner / phase        | Contents                                                        |
|-----------------|----------------------|----------------------------------------------------------------|
| `guides/`       | Backend (content, A) | `00-overview.md` + `lecture-NN-*.md` + `9N-exam-*.md` study guides — the embedded search corpus. See `guides/DOCUMENT.md`. |
| `mcqs/`         | Backend (content, B) | Curated `lecture-NN-*.json` MCQ files + `_taxonomy.md` — the quiz bank (500 questions). See `mcqs/DOCUMENT.md`. |
| `mcqs-archive/` | Backend (curation)   | The 20 archived original MCQ files (set-1 + set-2, 4,337 questions) replaced by the curated bank. **Not loaded by the app** (`quiz.py` globs `mcqs/` only). Restore = move files back into `mcqs/`. See `mcqs-archive/DOCUMENT.md`. |
| `chroma_db/`    | Generated by `ingest`| Persisted ChromaDB vector store. **Gitignored**; lives in the `chroma` volume in containers. |
| `slides/`       | Generated by `/slide`| Rendered slide-PNG disk cache. **Gitignored**; `slides` volume in containers. |

## Key decisions

- **Two data planes, one source-of-truth pattern.** Guides are markdown (human-
  authored, then embedded); MCQs are JSON (human-authored, validated, served from
  memory). Keeping them separate keeps the vector index clean and the quiz filters
  exact (architecture §5.3).
- **Numeric, zero-padded file naming.** Guides/MCQs use `lecture-NN-*` so directory
  listings sort correctly and the `L01…L12` join key is obvious.
- **Built artifacts are not committed.** `data/chroma_db/` and `data/slides/` are in
  `.gitignore`; they are rebuilt by `ingest` and on-demand rendering.

## How it connects

`ingest.py` reads `guides/` (env `GUIDE_ROOT`) and writes `chroma_db/` (env
`CHROMA_DIR`). `quiz.py` reads `mcqs/` (env `MCQ_ROOT`) into memory at app startup.
`/slide` writes `slides/` (env `SLIDES_DIR`). The compose file mounts `guides/` and
`mcqs/` read-only and maps `chroma_db/`/`slides/` to named volumes.
